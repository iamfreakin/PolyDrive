# 🌍 [TECHNICAL SPEC] WORLD & GRID SYSTEM
## 1. 월드 구조 (World Structure)
PolyDrive의 세계관은 **2차원 그리드(Local)**와 **도시 간 그래프(Global)**가 결합된 하이브리드 아키텍처로 구성됩니다.

---

## 2. 그리드 이동 시스템 (Local Grid)

### 2.1 좌표계 (Coordinate System)
- **맵 사이즈**: `40 x 40` 정수 좌표계.
- **경계 조건**: `(0, 0)` ~ `(39, 39)` 범위를 벗어나는 이동 시도는 시스템에 의해 차단됩니다.
- **플레이어 위치**: `MapManager`에 의해 실시간으로 추적되며, `X/Y` 정수값으로 관리됩니다.

### 2.2 렌더링 뷰포트 (Rendering Viewport)
- 성능과 가독성을 위해 콘솔에는 플레이어 주변의 일정 범위를 그리거나, 현재 프로젝트의 경우 `40x40` 전체를 콘솔 너비에 맞춰 렌더링합니다.

---

## 3. 도시 네트워크 (Global Graph)

### 3.1 노드(Node: City)
- 각 도시는 고유한 이름과 그리드 상의 고정 좌표(`X, Y`)를 가집니다.
- 예: `Gwangju (5, 35)`, `Seoul (15, 5)` 등.

### 3.2 엣지(Edge: Route)
- 도시 간 연결은 명시적인 `Route` 데이터에 의해 정의됩니다.
- **Distance**: 물리적 좌표 거리가 아닌, 실제 도로 상황을 반영한 가상 거리 데이터.
- **Reward**: 해당 경로 운송 시 지급되는 기본 보상금.

---

## 4. 데이터 지향 설계 (Data-Driven Design)

모든 월드 데이터는 `WorldData.h`에서 중앙 집중 관리됩니다.

- **CITIES**: 도시 리스트 및 좌표 정의.
- **ROUTES**: 도시 간 연결 관계 및 보상 밸런싱.
- **확장성**: 해당 데이터 리스트에 새 항목을 추가하는 것만으로 별도의 로직 수정 없이 월드 맵과 미션 시스템이 확장되는 **데이터 중심 설계**를 채택하고 있습니다.

---

## 5. 지리적 투영 (Geographic Projection)
대한민국의 지형적 특성을 고려하여 좌표를 배치하였습니다.
- **서부권**: 서울, 전주, 광주 (작은 X값)
- **동부권**: 강릉, 부산 (큰 X값)
- **북부권**: 서울, 강릉 (작은 Y값)
- **남부권**: 광주, 부산 (큰 Y값)
