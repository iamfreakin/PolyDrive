# 05. 게임 루프와 UI 분리 (Main.cpp, UIManager.cpp)

프로그램이 어떻게 살아 움직이는지, 그리고 왜 화면 출력과 로직을 분리해야 하는지 학습합니다.

## 1. 게임 루프 (Main Loop)
`Main.cpp`의 `while` 문은 게임이 종료될 때까지 무한히 반복됩니다.
1. **출력**: 현재 상태를 보여줌.
2. **입력**: 사용자의 명령을 기다림.
3. **처리**: 명령에 따라 `WorldManager`의 데이터를 변경함.

이것을 **Update-Render** 사이클이라고 부르며, 모든 게임의 근간이 됩니다.

## 2. 역할 분담 (Separation of Concerns)
- **WorldManager**: 게임의 "규칙"과 "데이터"를 담당합니다. (예: 돈이 충분한지, 에너지가 깎이는지)
- **UIManager**: 오직 "화면 출력"만 담당합니다. (예: 어떤 글자를 어디에 찍을지)

### 왜 나누나요?
나중에 텍스트 게임이 아니라 그래픽 게임으로 업그레이드하고 싶을 때, `WorldManager`는 건드리지 않고 `UIManager`만 교체하면 되기 때문입니다. 이것이 유지보수에 유리한 설계입니다.

## 3. 예외 처리 (Exception Handling)
```cpp
if (!(std::cin >> choice)) {
    std::cin.clear();
    std::cin.ignore(1000, '\n');
    continue;
}
```
사용자가 숫자가 아닌 문자를 입력했을 때 프로그램이 무한 루프에 빠지거나 종료되지 않도록 입력 버퍼를 비워주는 중요한 로직입니다. 교육용 코드에서 실수를 방지하는 필수적인 부분입니다.
