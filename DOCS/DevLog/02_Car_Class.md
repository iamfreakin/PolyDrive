# 02. 클래스 설계와 추상 클래스 (Car.h)

C++ 프로그래밍에서 **클래스(Class)**는 객체를 만들기 위한 설계도입니다. PolyDrive의 모든 차량은 `Car`라는 공통 설계도를 바탕으로 만들어집니다.

## 1. 클래스란 무엇인가?
클래스는 데이터(변수)와 그 데이터를 다루는 동작(함수)을 하나로 묶는 틀입니다.
- **멤버 변수**: 이름, 속도, 효율, 내구도 등 차량의 상태를 저장합니다.
- **멤버 함수**: 이동하기, 스펙 보여주기 등 차량의 동작을 정의합니다.

```cpp
// Car.h 핵심 구조
class Car {
protected:
    std::string name;
    float baseSpeed;
    float efficiency; 
    int durability;   
    int price;
    // ...
```

## 2. 접근 제어자: 내부 정보를 보호하기
`Car.h` 코드를 보면 변수들이 `protected`로 선언되어 있습니다.
- `private`: 나(해당 클래스)만 볼 수 있음.
- `protected`: 나 그리고 **나를 상속받은 자식들**만 볼 수 있음.
- `public`: 누구나 볼 수 있음.

> **설계 팁**: 자식 클래스(`Bus` 등)가 부모의 변수에 직접 접근하여 연산해야 하므로 `protected`를 사용했습니다.

## 3. 가상 함수와 가상 소멸자 (virtual)
```cpp
// Car.h 인터페이스
virtual ~Car() {} // 가상 소멸자
virtual float Move(float distance) { ... }
virtual void ShowSpec() const = 0; // 순수 가상 함수
```
- **다형성(Polymorphism)**을 구현하기 위한 핵심 키워드입니다. 부모 포인터(`Car*`)로 자식을 관리할 때, 실제 객체의 함수가 호출되도록 돕습니다.
- **가상 소멸자**: 부모 포인터로 객체를 지울 때 자식 클래스의 메모리까지 깨끗하게 지우기 위해 반드시 필요합니다.

### 추상 클래스(Abstract Class)란?
하나 이상의 **순수 가상 함수(`= 0`)**를 가진 클래스입니다. 우리 게임에서 "그냥 차"라는 것은 존재하지 않고, 반드시 "버스"나 "트럭"이어야 하므로 `Car`는 직접 객체를 만들 수 없는 **설계도 전용 클래스**로 존재합니다.
