# 04. 벡터와 메모리 관리 (WorldManager.cpp)

객체들을 저장하고 관리하는 **컨테이너**와 **메모리 관리**에 대해 학습합니다.

## 1. `std::vector` (동적 배열)
`vector`는 크기가 자유롭게 변하는 배열입니다. 우리 게임에서는 `Car*` 포인터를 담는 가방 역할을 합니다.
```cpp
std::vector<Car*> garage;
garage.push_back(newBus); // 가방에 새 차 넣기
garage.erase(it);         // 가방에서 차 빼기
```

### 왜 포인터(`Car*`)를 저장하는가?
다형성을 구현하기 위해서입니다. 그냥 `vector<Car>`라고 선언하면 자식 클래스들의 고유한 정보가 잘려나가는 **슬라이싱(Slicing)** 현상이 발생합니다. 자식 객체의 주소를 담아야만 나중에 그 객체가 버스인지 트럭인지 알 수 있습니다.

## 2. 동적 할당 (`new`와 `delete`)
- **`new`**: 힙(Heap) 메모리에 객체를 생성합니다. 이 메모리는 우리가 수동으로 지워주기 전까지 사라지지 않습니다.
- **`delete`**: `new`로 만든 객체를 메모리에서 해제합니다.

```cpp
WorldManager::~WorldManager() {
    for (auto car : garage) delete car; // 게임 종료 시 모든 차 해제
}
```

## 3. 팩토리 로직 (Factory Logic)
`GenerateShop()` 함수는 랜덤한 확률로 서로 다른 자식 객체(`Bus`, `Truck` 등)를 생성합니다. 이렇게 객체 생성을 담당하는 로직을 **팩토리(Factory)** 패턴이라고 부릅니다. 
사용자(플레이어)는 내부적으로 어떤 차가 어떻게 만들어지는지 몰라도, 상점에 나온 차를 보고 고르기만 하면 됩니다.
